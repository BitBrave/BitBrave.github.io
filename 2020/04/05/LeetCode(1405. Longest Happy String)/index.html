LeetCode(\1405. Longest Happy String)周赛第3题题解

A string is called happy if it does not have any of the strings 'aaa', 'bbb' or 'ccc' as a substring.

Given three integers a, b and c, return any string s, which satisfies following conditions:

  • s is happy and longest possible.
  • s contains at most a occurrences of the letter 'a', at most b occurrences of the letter 'b' and at most c occurrences of the letter 'c'.
  • swill only contain 'a', 'b' and 'c' letters.

If there is no such string s return the empty string "".

解题思路

Medium

给出abc三个数字分别代表abc三个字符的个数，从中选出尽可能多的字符组成一个字符串，字符串不能出现aaa这样的三个连一块儿的。返回最长的可以组成的字符串，答案有很多个，返回其中一个即可。

我的思路：最近算法有点渣···。这个题我第一想法是使用优先队列做Greedy，但是想了会没有想好怎么用。后来想到一个匹配办法，首先将abc三个数按照最小的依次取abc字符组成abcabc…直到其中某个字母的个数为0。然后将剩下的两个比如ab、bc之类的再排后边，然后就只剩下一个字母还有剩了，再插空就可以了。这样一定是可以的，但是代码太复杂了，我写不出来。

膜拜了大佬的算法，特此记录。

Greedy

一个简单的策略，假设a>=b>=c。我们在最终的字符串中尽可能选两个aa。如果选了两个aa之后a类型字母的个数还大于等于b，那么再选一个b。因为在下一次的递归中首先还是会选a所以要加入一个b作为缓冲。

直觉来看，这样的策略是合理的，我们每次都尽可能选多的字母做插入和排列。
