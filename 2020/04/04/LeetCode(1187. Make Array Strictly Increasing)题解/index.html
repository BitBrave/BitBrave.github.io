LeetCode(\1187. Make Array Strictly Increasing)题解

Given two integer arrays arr1 and arr2, return the minimum number of operations (possibly zero) needed to make arr1 strictly increasing.

In one operation, you can choose two indices 0 <= i < arr1.length and 0 <= j < arr2.length and do the assignment arr1[i] = arr2[j].

If there is no way to make arr1 strictly increasing, return -1.

解题思路

Hard

给定两个长度分别为n和m的数组arr1和arr2，可以将arr1中任何位置的数换为arr2的整数，现在要求使用最少的交换次数使得arr1中的元素严格递增，返回交换次数。如果不能完成，就返回-1。

本题使用DP，有点难，我也是参考了网上大神的解法才解出来的。

一般来说想到了dp，就需要设定状态转移方程，同时一般是从i通过状态转移方程逐步向后填充。

这里我们使用两个数组，或者说用两个函数来表示，

f(i,j)表示arr1的i-1元素与arr2的j-1元素交换的情况下，arr1中前0~i-1数字之间的最少交换次数。

g(i)表示arr1中的i-1元素不交换的情况下，arr1中前0-i-1数字之间的最少交换次数。

先不考虑如何填充f和g，那么最终算法的返回ret应该是，min(g(i), f(n-1,0), f(n-1,1), …, f(n-1,m-1)). 当然g(i)和f(n-1, *)必须是不为-1的才行。

这里为了遍历方便，我们可以将arr2先升序排序，并去掉重复的元素。假设还是arr2，长度为m。

那么如何来填充f和g。假设我们走到了第i个字符，f(i-1,*)和g(i-1)和之前都已经填充完毕。

  • i = 0，第一个字符，天然有序。f(0,*) = 1，g(0) = 0。
  • i > 0，f(i,*)=INT_MAX，g(i)=INT_MAX。确定了arr1[i]，尝试所有的arr1[i]和arr2[j]，j=0,…,m-1。
    1. 两个都不用替换。arr1[i] > arr1[i-1]，表示arr1当前字符可以不用换，g(i) = g(i-1);
    2. 换一个，看是否能替换arr1[i]。arr2[j] > arr1[i-1]，表示arr2中j当前数可以替换arr1[i]，f(i,j) = g(i-1) + 1;
    3. 换一个，看是否能替换arr1[i-1]。arr1[i] > arr2[j]，表示arr2[j]可以替换arr1[i-1]，g(i) = min(g(i), f(i-1,j));
    4. 查看两个是否都能换。查看所有arr2中小于arr2[j]的元素arr[k]，因为是严格升序，所以k<j。使用这两个元素一起替换arr1中第i和i-1个。f(i,j) = min(f(i,j), f(i-1,k)+1), k = 0,…,j-1。

这里在j从0~m-1的过程中，第4步可以提前使用一个变量记录上一层f(i-1,*)中前j-1分别与arr1[i-1]交换的最小值，这样在第4步就可以不用再依次向前遍历了。

这里默认值设为INT_MAX，可能会造成越界，因此在保证数目不会低于最差的交换次数，写小一点，m+1即可。

时空复杂度

代码如下，时间上，O(mn)+O(mlogm)，空间上，O(mn)，但因为状态转移只用到了上一层，所以可以更新到O(2m+n).
